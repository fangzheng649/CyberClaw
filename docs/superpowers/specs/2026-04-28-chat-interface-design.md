# CyberClaw 用户-AI 交互界面设计

> 日期: 2026-04-28
> 状态: 已确认
> 范围: 用户与 CyberAgent AI 的交互界面原型设计
> AI 后端策略: 竞赛演示阶段使用 **Mock AI 后端**（关键词匹配 + 预录响应），不连接真实 LLM。Mock 策略：用户输入匹配预定义关键词（如"扫描"触发 nmap 响应、"漏洞"触发 CVE 响应），返回模拟的流式分析结果。后续可替换为真实 OpenClaw Agent。
> 认证: MVP 阶段不实现认证，单用户模式。

## 1. 设计背景

CyberClaw 已完成 3D 安全 HUD（攻击可视化界面），但用户与 AI Agent 的交互界面尚未设计。本设计定义"对话页面"——管理员通过自然语言与 CyberAgent 交互的主要界面。

### 核心约束

- 目标用户：非安全专业的网络管理员
- 设计原则：看得见（3D 可视化）、听得懂（自然语言）、会动手（自动隔离修复）
- 技术栈：Web 前端（与 3D HUD 共享同一技术栈）
- 双页面架构：对话页和 3D HUD 页分开，避免性能干扰和界面拥挤
- 路由：同一 Express 服务器，两个 HTML 入口（`/` 为 HUD，`/chat` 为对话页）。Vite 开发模式下配置多入口
- 数据持久化：MVP 阶段使用服务端内存存储（对话历史、报告、操作日志），页面刷新后数据保留。不引入数据库
- 响应式：不支持移动端，专为桌面/投影仪场景设计
- 并发操作：同时只允许一个分析/操作流程执行，新请求排队等待前一个完成

## 2. 页面架构

### 2.1 双页面结构

系统由两个独立的浏览器标签页组成，共享 WebSocket 连接实现状态同步：

```
标签页 1: 3D HUD 监控页面（已实现）
  - 3D 拓扑可视化
  - 告警时间线
  - 设备详情面板
  - 底部迷你对话条（可折叠）

标签页 2: 对话页面（本设计）
  - 标签页导航：对话 | 报告 | 操作 | 历史
  - 固定底部输入栏
  - 右上角"3D HUD"快捷切换按钮
```

### 2.2 对话页面结构

```
┌─────────────────────────────────────────────────┐
│  CYBERCLAW  │ 对话 │ 报告 │ 操作 │ 历史 │ ↗ 3D HUD │
├─────────────────────────────────────────────────┤
│                                                 │
│              当前标签页内容区                      │
│                                                 │
├─────────────────────────────────────────────────┤
│  [输入框]                                  [发送] │
└─────────────────────────────────────────────────┘
```

- 输入栏固定在底部，在任何标签页都能直接输入（发送时自动切到"对话"标签，而非聚焦时）
- 右上角提供"3D HUD"按钮，一键切换到监控页面

## 3. 对话标签页

### 3.1 四种消息形态

AI 回复有 4 种核心消息形态：

**形态 1：简单问答**
- 直接的自然语言回复，关键数据高亮
- 示例：用户问"网络里有多少台设备？" → AI 回复"当前网络中共有 **15** 台设备，其中 **13** 台安全，**2** 台存在风险。"

**形态 2：多步分析（流式）**
- 渐进展示每步工具调用，显示 MCP 工具名标签
- 每步状态：✓ 完成 / ⟳ 进行中 / ○ 待执行
- 每步可展开查看原始工具调用参数和返回结果
- 示例流程：
  ```
  ✓ [nmap-scan] 网络扫描完成 — 发现 15 台设备
  ✓ [cve-intel] 漏洞查询完成 — 发现 2 个高危 CVE
  ⟳ [security-baseline] 正在执行安全基线检查...
    ○ [report] 生成分析报告
  ```

**形态 3：操作确认**
- 高风险操作（设备隔离、IP 封禁等）必须人工确认
- AI 提供建议方案，用户有 3 个选项：
  - **确认执行**：执行 AI 建议的操作
  - **暂不操作**：跳过此次操作
  - **修改方案**：通过对话调整操作参数
- 确认后实时显示执行结果

**形态 4：内嵌可视化**
- 攻击时间线、设备拓扑缩略图、统计图表嵌入对话消息中
- 提供"在 3D HUD 中查看"链接，点击跳转到 HUD 页面的完整版本

### 3.2 对话消息样式

- 用户消息：右对齐，绿色边框，深色背景
- AI 消息：左对齐，"CyberAgent" 标签，带状态指示灯
- MCP 工具标签：小型标签形式显示在分析步骤中，如 `[nmap-scan]`
- 操作确认框：红色警告边框，独立卡片区域
- 内嵌可视化：深色背景卡片，带边框，可交互

## 4. 报告标签页

### 4.1 布局

左右分栏：
- **左侧**：报告列表，按日期排序，显示标题 + 生成时间
- **右侧**：选中报告的完整内容

### 4.2 报告类型

顶部过滤标签：
- 全部报告
- 安全巡检
- 攻击复盘
- 合规审计

### 4.3 报告内容结构

```
报告标题 + 生成时间 + 元信息
├── 统计卡片（设备总数/安全设备/存在风险/CVE漏洞）
├── 关键发现列表（按严重度着色）
│   ├── CRITICAL: 红色，CVE 详情
│   └── WARNING: 黄色，风险描述
├── 详细分析（按设备/按类型）
└── 操作建议
```

### 4.4 导出

支持 PDF 导出和打印。

报告来源：AI 从对话中自动生成（当分析完成时），或用户手动请求"生成安全报告"。

## 5. 操作标签页

### 5.1 布局

左右分栏：

**左栏：**
- 快捷操作按钮（2×2 网格）：
  - 全网扫描
  - 漏洞检测
  - 设备隔离
  - 生成报告
- MCP 服务器状态列表（显示所有已注册的服务器，数量随部署变化）：
  - 显示在线/忙碌/离线状态
  - 可展开查看最后调用时间和结果

**右栏：**
- Skills 列表：标签形式展示，按类型颜色区分（感知/检测/响应/复盘）
- 操作执行历史（按时间倒序）：
  - 每条记录包含：时间戳、操作描述、执行结果
  - 按操作类型着色（绿色=成功，红色=失败，蓝色=信息）
  - 点击可展开查看完整的 CLI 输出和 MCP 调用链

### 5.2 快捷操作流程

点击快捷按钮后：
1. 切换到对话标签页
2. 自动填充对应的自然语言指令
3. 用户可修改或直接发送
4. AI 执行并流式展示结果

## 6. 历史标签页

### 6.1 布局

左右分栏：
- **左侧**：历史会话列表，按日期分组，显示会话摘要和关键统计
- **右侧**：选中会话的完整对话记录

### 6.2 功能

- 关键词搜索历史对话
- 按日期范围过滤
- 导出选项（文本/JSON）

## 7. WebSocket 协议 & 消息格式

### 7.1 连接

对话页和 HUD 页各自独立连接 `ws://localhost:3001/ws`，服务端向所有客户端广播事件。

### 7.2 客户端发送消息

```json
{
  "type": "chat_message",
  "content": "帮我检查网络中所有 IoT 设备的安全状态",
  "session_id": "session-uuid"
}
```

### 7.3 服务端响应格式

**简单回复：**
```json
{
  "type": "chat_reply",
  "message_id": "msg-uuid",
  "content": "当前网络中共有 **15** 台设备...",
  "reply_type": "simple"
}
```

**多步分析（流式）：**
```json
{
  "type": "chat_step",
  "message_id": "msg-uuid",
  "step_index": 0,
  "step_total": 4,
  "tool": "nmap-scan",
  "status": "completed",
  "summary": "网络扫描 — 15台设备",
  "detail": { "devices": 15, "ports_open": 30 }
}
```

每个步骤独立推送，前端按 `message_id` 聚合到同一消息气泡中。

**操作确认：**
```json
{
  "type": "chat_confirmation",
  "message_id": "msg-uuid",
  "title": "隔离操作确认",
  "actions": [
    { "label": "确认隔离", "action": "confirm", "style": "danger" },
    { "label": "暂不操作", "action": "cancel", "style": "default" },
    { "label": "修改方案", "action": "modify", "style": "info" }
  ],
  "details": "关闭 Switch-Core Gi0/1 端口..."
}
```

用户点击按钮后发送：
```json
{
  "type": "chat_confirmation_response",
  "message_id": "msg-uuid",
  "action": "confirm"
}
```

**内嵌可视化：**
```json
{
  "type": "chat_reply",
  "message_id": "msg-uuid",
  "reply_type": "visualization",
  "viz_type": "timeline",
  "content": "攻击时间线回放：",
  "viz_data": {
    "events": [
      { "time": "T+0s", "label": "系统上线", "severity": "info" },
      { "time": "T+25s", "label": "暴力破解成功", "severity": "critical" }
    ]
  }
}
```

### 7.4 错误处理

| 场景 | 前端行为 |
|------|---------|
| WebSocket 断开 | 显示"连接断开，正在重连..."横幅，自动重连（指数退避，最多 5 次） |
| AI 响应超时（>30s） | 在消息中显示"分析超时，请重试"并提供重试按钮 |
| MCP 工具调用失败 | 步骤标红，显示错误信息，AI 生成降级建议 |
| 流式中断 | 保留已收到的步骤，标记"分析中断"，提供"继续分析"按钮 |
| 操作确认超时（>5min） | 自动取消，通知用户"操作已超时取消" |
| 并发请求冲突 | 显示"当前有分析正在进行，请等待完成"提示 |

## 8. 跨页面联动

### 8.1 联动机制

两个页面各自独立连接同一个 WebSocket 端点（`ws://localhost:3001/ws`），服务端广播事件自然双页同步。`BroadcastChannel` API 仅用于客户端主动跨标签页操作（如"在 HUD 中聚焦设备 X"）。

### 8.2 联动行为

**对话页 → HUD：**
- 点击设备名/IP → 自动切换到 HUD 标签页并聚焦该设备
- 点击内嵌可视化中的"在 3D HUD 查看" → 跳转
- 执行安全操作时 → HUD 同步更新设备状态

**HUD → 对话页：**
- 设备详情面板增加"问 AI"按钮 → 切换到对话页并带上设备上下文
- 告警卡片增加"追问"按钮 → 切换到对话页并追问详情
- 底部迷你输入条（可折叠）→ 无需跳转即可快速提问

**双向状态同步：**
- 设备状态变更同时反映在两个页面
- 操作执行结果同步展示
- WebSocket 事件双页面共享

### 8.3 HUD 迷你对话条

在 3D HUD 页面底部，增加一个可折叠的对话条：
- 默认收起为一行输入框
- 展开后显示最近 3 条 AI 回复
- 输入问题后自动发送到 AI，回复在展开区域显示
- 复杂问题显示"在对话页查看完整回复"链接

## 9. 数据流

```
用户输入自然语言
    ↓
OpenClaw Agent（AI 决策层）
    ├── 识别意图 → 选择 Skill
    ├── Skill 编排调用 MCP 工具
    │     ├── nmap-scan → 返回扫描结果
    │     ├── cve-intel → 返回漏洞信息
    │     └── auto-response → 执行安全操作
    ├── AI 综合分析
    └── 生成自然语言回复
    ↓
对话页展示（流式）+ HUD 同步更新
```

## 10. 关键交互规格

| 交互 | 触发 | 结果 |
|------|------|------|
| 自然语言提问 | 底部输入框发送 | AI 流式回复，对话标签页 |
| 多步分析 | 复合问题触发 Skill | 渐进展示工具调用链 |
| 操作确认 | AI 建议高风险操作 | 内联确认卡片，三个按钮 |
| 快捷操作 | 操作页按钮点击 | 跳转对话页，预填指令 |
| 设备聚焦 | 对话中点击设备名 | 跳转 HUD 页，聚焦设备 |
| 迷你提问 | HUD 底部输入框 | HUD 内展开回复，无需跳转 |
| 报告查看 | 报告标签页 | 左右分栏浏览，支持导出 |
| 历史搜索 | 历史标签页 | 按关键词搜索历史对话 |
