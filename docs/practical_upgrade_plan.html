Suricata 真实流量检测集成

P0
当前问题 CyberClaw 的"检测"完全依赖外部扫描和 syslog 上报，没有 DPI 能力。无法检测正在发生的网络攻击（端口扫描、暴力破解、异常流量）。
方案：集成 Suricata + scapy fallback 嵌入 Suricata 作为 IDS 引擎，后端 tail eve.json 实时解析告警，映射到 5-state FSM。Suricata 不可用时降级到 scapy 流量统计。

FSM 状态映射

Suricata Severity典型场景CyberClaw FSM
1 (Critical)恶意软件检测、Shellcodeattacked
2 (High)Web 攻击、暴力破解成功attacked
3 (Medium)端口扫描、可疑连接vulnerable
4 (Low)信息性告警scanning

实施步骤

  1. 创建 server/services/suricata_service.py：SuricataMonitor 类 — tail eve.json + 告警解析 + FSM 映射 + scapy fallback。单例模式 get_suricata_service()
  2. 修改 server/main.py：导入 suricata_service，wire broadcast callback，lifespan 中启动监控
  3. 修改 server/api/tools.py：新增 /suricata/start、/stop、/alerts、/stats 四个端点
  4. 新增 WebSocket 消息：suricata_alert（告警推送）、traffic_stats（流量统计）、device_status_change（FSM 变更）
不新增 Python 依赖。使用标准库 open() + 轮询替代 aiofiles。Suricata 可选安装，不安装时不 crash，返回空数据。

server/main.py MOD +10行
server/api/tools.py MOD +40行
投入：中等 — 核心约 200 行 Python，Suricata 规则集另配

OpenClaw 接入评估

评估
评估结论 OpenClaw 是基础设施，不是约束。 CyberClaw 已在使用 openclaw.json 注册 MCP 服务器，这已是最核心的接入点。新功能在 CyberClaw 原生代码中实现，不强行适配 OpenClaw 模式。

OpenClaw 能力 vs CyberClaw 状态

OpenClaw 能力CyberClaw 状态结论
MCP Server 注册 (openclaw.json)已在使用继续使用
Agent 工作区管理未使用不需要 — 有自己的 FastAPI
Token 优化 (TOON 序列化)配置了但未使用可选 — 当前 GLM-4 成本低
安全模式 (hobby/production)hobby 模式可选 — 当前无敏感数据
Skills 框架未使用不需要 — 有 tool_broadcast
GAIT 日志未使用有价值，非 P0

接入策略

1. 继续使用 openclaw.json 注册所有 MCP 服务器（包括可能的 suricata-analyzer）

2. 不创建新 MCP 服务器 — Suricata 监控是持续运行的服务，不是按需调用的工具。MCP stdio 不支持持续数据流

3. 不引入 OpenClaw Agent 框架 — CyberClaw 有自己的 FastAPI backend + WebSocket 架构，更灵活

4. 未来可选：当需要接入 Claude Agent 时，可以通过 OpenClaw 的 agent 模式扩展

config/openclaw.json MOD +10行 (可选)
投入：极低 — 仅配置文件更新，无代码改动

验证方案

无 Suricata 时（当前环境）

  1. 启动后端 python -m uvicorn server.main:app --reload --port 8000
  2. 启动前端 cd ui/cyberclaw-hud && npm run dev
  3. 打开 http://localhost:3000 → 验证统一界面正常（左 HUD + 右 Chat/Dashboard）
  4. 点击 Dashboard Tab → 验证告警列表显示"暂无告警"（不是 crash）
  5. curl /api/dashboard/alerts → 返回空列表 {"alerts":[],"total":0}
  6. curl /api/dashboard/trends/device-status → 返回设备 FSM 状态分布
  7. 触发 Mirai 演示 → 验证 HUD 动画 + Dashboard 告警同步更新

有 Suricata 时（WSL2）

  1. WSL2: sudo apt install suricata && sudo suricata-update
  2. 启动: sudo suricata -c /etc/suricata/suricata.yaml -i eth0
  3. 触发扫描: nmap -sS <target>
  4. 验证: HUD 设备状态变为 attacked/vulnerable
  5. 验证: Dashboard 告警列表实时更新 + ECharts 图表显示趋势
  6. 验证: Chat 可正常对话，不受 Suricata 影响
